File formats

All external files should begin with START and end with STOP. This minor annoyance is to allow IE running on Macs to properly read the file - Explorer will attempt to read the resource fork in addition to the data, resulting in junk being parsed.

Blank lines may be included for legibility.

Chromosome Data Files

These files contain instructions to specify a virtual chromosome and marker set to be tested. Begin each chromosome with > (greater than), followed immediately by a name for that virtual chromosome. The size of the chromosome is then specified after a tab. Next list the markers on the chromosome: the name of the marker, tab, and then its position. Positions can be designated as Mb, Kb or bp. The program will convert all values to kb for consitancy. That's it!

An example, specifying two chromosomes, each 30Mb long. One has two markers seperated by 20kb. The other is similar, but has a duplication of marker B about 10Mb from the first pair. 

START

>A-020-B  TAB  30Mb
A  TAB  10Mb
B  TAB  10.02Mb

>A-020-B---B  TAB  30Mb
A  TAB  10Mb
B  TAB  10.02Mb
B  TAB  20Mb

STOP

Statistics Data Files

These files contain statistical distributions for virtual chromosomes analyzed by the program. In general, you should construct these just by collating the output files produces when you run a stepped simulation.
